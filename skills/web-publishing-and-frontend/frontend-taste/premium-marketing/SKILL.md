# Premium Marketing / Landing Pages

A sub-skill of [frontend-taste](../SKILL.md). The core rules still apply; this layers the specifics for a page that has to sell and feel expensive without sliding into template-builder cliché.

## When to use this skill

Use it, routed from frontend-taste, when the task is conversion-driven and meant to impress: a product launch page, a landing page, a pricing page, a homepage selling a product or service. Do not use it for reading-first content (use `minimalist-editorial`) or internal data views (use `data-dense-dashboard`).

## Steps

### Step 1 — Design the section rhythm, not a stack of cards

Plan the page as a sequence of distinct sections with deliberate variance in layout: the hero, the proof, the detail, the close should not all be the same shape. Never the default hero-plus-three-cards. Completion: each major section has a different, intentional structure.

### Step 2 — Build one confident hero

The hero earns the most space, the largest type, and the single clearest call to action. One headline, one supporting line, one primary button using the accent. Resist a second competing CTA above the fold. Completion: the hero has one unmistakable primary action.

### Step 3 — Make the type and spacing feel premium

Use the largest steps of the type scale for display moments and generous whitespace to signal quality; cramped layouts read cheap. A premium feel comes from scale, space, and restraint, not from more effects. Completion: display type and section spacing carry a sense of quality.

### Step 4 — Spend motion and accent sparingly

One restrained accent for primary actions and key emphasis, neutral elsewhere. Motion supports attention (a subtle reveal, a considered hover) and is never decoration for its own sake. No purple-gradient-on-white. Completion: accent and motion are deployed on purpose, not everywhere.

### Step 5 — Run the frontend-taste verification loop at multiple viewports

Screenshot the full page at desktop and mobile widths. Inspect for section monotony, weak hero, cramped spacing, and overused accent. Fix the weakest, repeat. Completion: the loop's last pass at both viewports finds nothing weak.

## Guardrails

- Never build the page as a uniform stack of identical card sections; deliberate section variance is the requirement.
- Never reach for a purple-gradient-on-white, multiple competing CTAs above the fold, or motion that exists only as decoration.

## Output contract

A conversion-driven page built as a sequence of deliberately varied sections, one confident hero with a single primary CTA, premium type and spacing, a restrained accent and purposeful motion, plus verification screenshots at desktop and mobile widths.

## Verification standard

Do not call it done until: sections vary deliberately, the hero has one clear primary action, display type and spacing read premium, the accent and motion are restrained, and the frontend-taste loop ran on desktop and mobile screenshots with no remaining weakness.
